# 용어집

> **TL;DR** — 이 저장소에서 쓰는 도메인·기술 용어의 뜻을 정리합니다.

## 도메인

- **진료과 (department)** — 안과·내과·소아과 같은 병원의 과. `data.json`의 `departments` 배열 한 항목에 대응합니다.
- **대기 시간 (waitMinutes)** — 해당 진료과의 예상 대기 시간(분). 화면에는 "대기 시간 약 N분"으로 표시.
- **대기 인원 (waitingCount)** — 해당 진료과에서 대기 중인 환자 수(명). 모의 폴링이 진행될 때마다 1씩 줄어듭니다.
- **대기자 명단 (waitingPatients)** — 해당 진료과에서 대기 중인 환자 이름 배열. 카드를 펼치면 표시되며, 화면에는 마스킹되어 나옵니다.
- **이름 마스킹 (maskName)** — 개인정보 보호를 위해 이름의 첫·끝 글자만 남기고 가운데를 `*`로 가리는 처리. 예: `김민수` → `김*수`.
- **현재 호출 번호 (currentNumber)** — 진료과에서 지금 호출 중인 대기 번호. 내 대기 번호에서 이 값을 빼 내 순서를 계산합니다.
- **대기 번호 (ticketNumber)** — 사용자가 알림 신청 시 입력하는 자신의 번호. `currentNumber`보다 커야 유효합니다.
- **내 순서 (position)** — `ticketNumber - currentNumber`. 내 앞에 남은 호출 수이며, 3 이하가 되는 첫 시점에 알림이 발동합니다.
- **마지막 갱신 시각 (updatedAt)** — 데이터가 만들어진 시각. ISO 8601 문자열이며 헤더에 한국어 형식으로 표시됩니다.

## 기술

- **데이터 소스 (DATA_SOURCE)** — `app.js` 최상단 상수. 데이터를 가져올 위치를 가리킵니다. 현재는 `./data.json`이고, 실제 백엔드 연동 시 이 한 줄을 엔드포인트 URL로 교체합니다.
- **빌드리스 정적 사이트 (build-less)** — 번들러·패키지 매니저·서버 코드 없이 정적 파일만으로 동작하는 구성. 브라우저로 바로 열거나 단순 정적 서버로 띄웁니다.
- **카드 (card)** — 한 진료과의 대기 정보를 담은 화면 단위(`.department-card`). `app.js`가 진료과마다 하나씩 생성합니다. 클릭/키보드로 대기자 명단을 펼칠 수 있는 토글 버튼(`role="button"`, `aria-expanded`)입니다.
- **알림 신청 / 구독 (notifySubscriptions)** — 사용자가 대기 번호를 등록해 순서 임박 알림을 받는 것. `app.js`의 `Map`(키=진료과 id, 값=`{ ticketNumber, alerted }`)에 세션 동안만 보관되며 새로고침하면 사라집니다.
- **모의 폴링 (mock polling)** — 정적 `data.json`이 스스로 줄지 않으므로, `setInterval`(`POLL_INTERVAL_MS` = 3초)로 `currentNumber`를 올리고 `waitingCount`를 내려 호출 진행을 흉내 내는 클라이언트 시뮬레이션.
